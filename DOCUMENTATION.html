1. Introduction & Architecture

The Airfoil Visualizer is a full-stack web application that lets users explore airfoil aerodynamics interactively. Users can select predefined airfoils, generate NACA 4-digit airfoils, upload custom coordinate files, and adjust flight parameters to see real-time changes in lift, drag, and stall behavior.

The application follows a client-server architecture with a React frontend and an Express.js backend, communicating via a REST API.

5. Aerodynamic Formulas

All formulas are implemented in backend/utils/aerodynamics.js. These are simplified educational models, not CFD-grade simulations.

Lift Coefficient (Cl)

Cl = clAlpha × (α - α₀) × fRe_lift

Where:

  • clAlpha = 0.1 per degree — lift curve slope (from thin airfoil theory: 2π per radian ≈ 0.1097 per degree; NACA TN 3361 gives ~0.105 for NACA 0012 at Re=1.8×10⁶)
  • α = angle of attack (degrees)
  • α₀ = zero-lift angle of attack (degrees; negative for cambered airfoils)
  • fRe_lift = Reynolds number lift correction factor (see §7)

Zero-Lift Angle (α₀) — Thin Airfoil Theory

α₀ = -1/π ∫₀^π (dyc/dx)(cos θ - 1) dθ

where x = (1 - cos θ)/2 maps the chord to an angular coordinate, and yc(x) is the camber line. The integral is evaluated numerically via finite differences along the camber line.

Source: Glauert, H., "The Elements of Aerofoil and Airscrew Theory", Cambridge University Press, 1926, Chapter VII.

Stall Model

When α exceeds the effective stall angle (which is reduced at low Re), Cl decays exponentially:

Clpost-stall = Clmax × exp(-0.1 × (α - αstall,eff))

The stall angle baseline comes from published wind tunnel data:

AirfoilBaseline StallSource
NACA 001214° at Re 1.8×10⁶Critzos et al., NACA TN 3361 (1955)
NACA 241214° at Re 2.2×10⁶NASA TM (stall at α=14.4°)
NACA 441214° at Re 5×10⁵IJRMEET wind tunnel study

Drag Coefficient (Cd)

Cd = (Cdbase + Cdinduced + Cdthickness) × fRe_drag + Cdaoa_rise + Cdstall_rise

Components:

  • Base drag: Cdbase = 0.002 + Cf, where Cf = 0.074/Re1/5 is the Prandtl-Schlichting turbulent skin friction coefficient for a flat plate
  • Induced (lift-related) drag: Cdinduced = 0.01 × Cl² — simplified model of lift-induced drag (full equation: Cdi = Cl²/(π·AR·e))
  • Thickness drag: Cdthickness = maxThickness × 0.05 — form drag proportional to thickness ratio
  • AoA-dependent rise: When α > 12°, Cd multiplies by (1 + (α-12)×0.05)
  • Post-stall rise: When α > αstall,eff, Cd multiplies by (1 + (α-αstall,eff)×0.15)

Force Calculation

L' = ½ ρ V² c Cl   (lift per meter of span, N/m)
D' = ½ ρ V² c Cd   (drag per meter of span, N/m)

Where ρ = air density, V = velocity, c = chord length.

Lift-to-Drag Ratio

L/D = Cl / Cd

6. NACA 4-Digit Coordinate Generation

Implemented in backend/utils/nacaGenerator.js. The NACA 4-digit series defines airfoils by three parameters:

  • m — maximum camber (percent of chord, first digit)
  • p — position of max camber (tenths of chord, second digit)
  • t — maximum thickness (percent of chord, last two digits)

Camber Line

yc = (m/p²)(2px - x²)   for 0 ≤ x ≤ p
yc = (m/(1-p)²)(1 - 2p + 2px - x²)   for p ≤ x ≤ 1

Thickness Distribution

The half-thickness at any chordwise position x is:

yt = 5t(0.2969√x - 0.1260x - 0.3516x² + 0.2843x³ - 0.1015x⁴)

This polynomial was derived by NACA to match the maximum thickness t at approximately x=0.3c for all airfoils. The coordinates of the upper and lower surfaces are then:

xu = x - yt sin θ     yu = yc + yt cos θ
xl = x + yt sin θ     yl = yc - yt cos θ

where θ = arctan(dyc/dx). The generator produces 150 points per surface (301 total including TE) evenly spaced in x, with cosine clustering near LE and TE.

For NACA airfoils in airfoils.json, the coordinates are stored at 150-point resolution. For unknown NACA airfoils, the backend generates them on the fly at 100-point resolution.

7. Reynolds Number Corrections

The Reynolds number is:

Re = ρ V c / μ

where μ = 1.81×10⁻⁵ Pa·s (dynamic viscosity of air at 15°C).

αstall,eff = αstall,base - 4.0 × exp(-Re / 180000)

8. Data Flow

Simulator workflow

  1. User selects an airfoil from the dropdown (predefined or custom)
  2. useEffect triggers runCalculation() with a 300ms debounce
  3. Frontend sends POST to /api/calculate with airfoil name + 4 parameters
  4. Backend calculate controller validates inputs, calls findAirfoil to look up or generate the airfoil, then calls calculateForPredefined or calculateForCustom
  5. Aerodynamics module computes Cl, Cd, forces, Re, stall warning, and explanation text
  6. Response sent back as JSON; frontend updates state → re-renders results grid, SVG, and graphs

NACA Generator workflow

  1. User adjusts sliders (m, p, t) on the NACA Generator page
  2. Frontend generates coordinates locally using nacaGenerator.js utility (client-side copy of the formula)
  3. SVG preview updates in real time
  4. Clicking "Use in Simulator" saves to local storage and navigates to /simulator?naca=...&coords=...
  5. Simulator reads URL params, saves as a custom airfoil, and selects it

Custom airfoil upload workflow

  1. User uploads a .dat/.txt/.csv file or pastes coordinate text
  2. Frontend sends file/text to /api/airfoil/custom or /api/airfoil/upload
  3. Backend parses coordinates, computes geometry (max thickness, camber, stall estimate, zero-lift angle)
  4. Response includes coordinates + geometry; frontend saves to browser's localStorage
  5. User can then select it in the Simulator dropdown

9. Adding New Airfoils

Method A: Add to airfoils.json

Add a new entry with the following fields. All fields are optional — missing ones will be auto-computed from coordinates:

{
  "name": "Your Airfoil Name",
  "type": "Symmetric",
  "camberDescription": "Brief description",
  "maxThickness": 0.12,
  "maxThicknessLocation": 0.3,
  "maxCamber": 0.0,
  "maxCamberLocation": 0.0,
  "zeroLiftAngle": 0.0,
  "stallAngle": 14.0,
  "baseDragCoefficient": 0.006,
  "description": "A longer description shown in the simulator.",
  "coordinates": [
    {"x": 1.0000, "y": 0.0000},
    ...  (at least 5 points, x in [0,1])
  ]
}

If metadata is missing, the backend's computeAerodynamicProperties() in airfoilParser.js will:

  • Compute zero-lift angle from thin airfoil theory (numerical integration of camber line slope)
  • Estimate stall angle from camber + thickness (empirical: symmetric ~14°, cambered ~14°, very thick/thin adjusted)
  • Compute max camber and thickness from coordinate bounding

Method B: Upload via the Custom Airfoil page

Accepts coordinate files with 2+ columns separated by spaces, commas, or semicolons. Lines starting with non-numeric headers are skipped. The parser also strips surrounding quotes and UTF-8 BOM.

Error messages now include the specific line number for each parsing failure, e.g.:
"Line 12: x=2.5 is out of range [0, 1]"
